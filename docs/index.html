Overview

DeepDive presents an automated approach for training deep search agents that can navigate complex, multi-step information-seeking tasks. Our method combines automated data synthesis from knowledge graphs with end-to-end multi-turn reinforcement learning to create agents capable of sophisticated long-horizon reasoning and web browsing.

  • Automated Deep Search Data Synthesis — generate challenging QA pairs from knowledge graphs through controlled random walks.
  • Multi-Turn RL Training for Browsing — end-to-end reinforcement learning for deep search capabilities.
  • Test-Time Scaling — supports scaling via tool calls and parallel sampling.

Method

Stage 1 · Automated Data Synthesis from Knowledge Graphs

We synthesize complex, hard-to-find questions from open knowledge graphs in three steps:

  1. Knowledge Graph Random Walks. Starting from an initial node, navigate the graph for k steps to form a path. We use longer paths (k > 5) to increase reasoning complexity.
  2. Entity Obfuscation. Combine each node with its attributes into an attribute-rich path, then have an LLM generalize specific details into “blurry entities” that require deep search to resolve.
  3. Difficulty Filtering. A frontier model (GPT-4o) with basic search attempts each question four times; only questions it fails on all attempts are retained.

Stage 2 · End-to-End Multi-Turn Reinforcement Learning

At step t, the agent generates chain-of-thought ct, executes browsing action at, and observes web content ot. We train with Multi-Turn GRPO using group-relative normalized advantages and strict binary rewards: a trajectory earns +1 only if both format correctness and answer accuracy are satisfied. This prevents reward hacking.

Generalization on Simple Search Tasks

DeepDive-32B (SFT and RL) consistently outperforms strong baselines on simpler benchmarks (HotpotQA, Frames, WebWalker), achieving >60 points on WebWalker, surpassing WebShaper-72B (52.2).

Test-Time Scaling

Tool Call Scaling

Allowing DeepDive more tool calls during inference leads to higher accuracy on complex, multi-step tasks. On BrowseComp, accuracy rises from 8% at 8 tool calls to 15% at 128 tool calls. DeepDive-32B consistently outperforms its SFT-only variant, especially beyond 32 tool calls.

Parallel Sampling

DeepDive generates 8 independent reasoning trajectories per question in parallel. Selecting the answer with the fewest tool calls among 8 samples raises accuracy from 12.0% (single-shot) to 24.8% — more than doubling performance. Majority voting helps (18.8%), but is outperformed by minimal tool-call selection.

Additional Study: Semi-Automated i.i.d. Deep Search QA for RL

Training with i.i.d. data brings larger improvements. The 32B-RL model reaches 22.2% accuracy on BrowseComp, up from 14.8%.
