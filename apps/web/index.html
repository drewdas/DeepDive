Overview

DeepDive presents an automated approach for training deep search agents that can navigate complex, multi-step information-seeking tasks. The method combines automated data synthesis from knowledge graphs with end-to-end multi-turn reinforcement learning to create agents capable of sophisticated long-horizon reasoning and web browsing.

Method Overview

Stage 1 · Automated Data Synthesis from Knowledge Graphs

We synthesize complex, hard-to-find questions from open knowledge graphs in three steps:

  1. Knowledge Graph Random Walks. Starting from an initial node $v_0$, we navigate the graph for $k$ steps to form a path $P=[v_0, v_1, \ldots, v_k]$, where each step $(v_i, v_{i+1})$ is a valid edge. We choose longer path lengths ($k > 5$) to increase reasoning complexity.
  2. Entity Obfuscation. We combine each node $v_i$ with its attributes to form an attribute-rich path: $$P_A = [(v_0, [a_0^0, a_0^1, \ldots]), (v_1, [a_1^0, a_1^1, \ldots]), \ldots, (v_k, [a_k^0, a_k^1, \ldots])]$$ An LLM then obfuscates information along the entire path, creating "blurry entities" that require deep search to resolve.
  3. Difficulty Filtering. We use a frontier model (GPT-4o) with basic search to attempt each question four times. Only questions the frontier model fails in all attempts are retained.

Stage 2 · End-to-End Multi-Turn Reinforcement Learning

At step $t$, the agent generates chain-of-thought $c_t$, executes browsing action $a_t$, and observes web content $o_t$.

Multi-Turn GRPO Training. We employ Group Relative Policy Optimization with normalized advantages:

$$A_i = \frac{r_i - \text{mean}(\{r_k\}_{k=1}^G)}{\text{std}(\{r_k\}_{k=1}^G)}$$

Strict Binary Rewards. A trajectory receives reward +1 if and only if both format correctness and answer accuracy are satisfied:

$$ r(\mathcal{T}) = \begin{cases} 1, & (\forall i, \text{Format}(c_i, a_i)) \wedge \text{Judge}(a_{\text{eos}}, a^*) \\ 0, & \text{otherwise} \end{cases} $$

This strict reward mechanism ensures high-quality trajectories and prevents reward hacking.

Generalization on Simple Search Tasks

DeepDive-32B (SFT and RL) consistently outperforms strong baselines on simpler benchmarks such as HotpotQA, Frames, and WebWalker, achieving more than 60 points on WebWalker — surpassing WebShaper-72B (52.2).

Test-Time Scaling

DeepDive demonstrates remarkable test-time scaling through two mechanisms.

Tool Call Scaling

Allowing DeepDive more tool calls during inference leads to higher accuracy on complex, multi-step tasks. On BrowseComp, accuracy improves from 8% at 8 tool calls to 15% at 128 tool calls. DeepDive-32B consistently outperforms its SFT-only variant — especially when the allowed tool calls exceed 32 — indicating that the RL stage better equips the model to use long tool-call horizons.

Parallel Sampling

Beyond a larger tool-call budget, DeepDive leverages parallel sampling: for each question, the model generates 8 independent reasoning trajectories. Three selection strategies are considered — single-shot, majority voting, and choosing the answer that required the fewest tool calls.

Empirical analysis reveals that answers submitted earlier with fewer tool calls are usually more accurate. Selecting the answer with the minimal tool calls among 8 samples raises accuracy from 12.0% (single-shot) to 24.8% — more than doubling performance. Majority voting also helps (18.8%), but is outperformed by minimal tool-call selection.

Additional Study · Semi-Automated i.i.d. Deep Search QA for RL

We create a semi-automated framework for generating i.i.d. QA pairs. Training with i.i.d. data brings larger improvements: the 32B-RL model reaches 22.2% accuracy on BrowseComp (up from 14.8%), with improvements on Chinese benchmarks as well.
